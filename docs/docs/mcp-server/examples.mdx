Here are some concrete prompts you can use with your MCP client to interact with RushDB.

## Recommended starting prompt

Begin every session with:

> Call `getOntologyMarkdown` to load the full database schema.

This gives the agent all label names, field names, value ranges, and relationships before it makes any other call.

## Listing labels

> Ask the RushDB MCP server to run `findLabels` with `limit=20`.

## Creating and querying records

> Use `createRecord` to add a `Task` with data `{"title": "Write docs", "status": "open"}`.

> Now use `findRecords` with `where={"status": "open"}` and `orderBy={"createdAt": "desc"}` limit 5.

## Aggregation (counts, sums, grouping)

Before building an aggregation query, load the spec:

> Call `getSearchQuerySpec`, then run `findRecords` with `labels=["Order"]`, `aggregate={"total":{"fn":"sum","field":"amount"}}`, and `groupBy=["status"]`.

**Important:** never include `limit` when `aggregate` is present — it restricts the record scan and produces incorrect totals.

## Filtering with dates

> Call `getSearchQuerySpec` to review date operator syntax, then find all `Event` records where `startDate` is after `2025-01-01`.

## Setting records

> Use `setRecord` for `recordId="<id>"` with label `Task` and data `{"title": "Polish docs", "status": "in-progress"}`.

## Attaching relationships

> Call `attachRelation` from `sourceId="<sourceId>"` to `targetIds=["<target1>", "<target2>"]` with `relationType="references"`.

## Upsert by field

> Call `createRecord` with `label="User"`, `data={"email":"a@b.com","name":"Ann"}`, and `options={"mergeBy":["email"],"mergeStrategy":"append"}` to upsert by email.

## Bulk import with upsert

> Call `bulkCreateRecords` for `label="Product"` with `data=[{"sku":"A1","price":9.99},{"sku":"A2","price":14.99}]` and `options={"mergeBy":["sku"]}`.

## Export to CSV

> Call `exportRecords` with `labels=["Task"]` and `where={"status":"done"}` to export completed tasks.
